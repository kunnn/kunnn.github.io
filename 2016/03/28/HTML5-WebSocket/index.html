Html5之WebSocket

发表于 2016-03-28   |   分类于 Html5   |  

WebSocket是什么

WebSocket是一种用于在服务器与浏览器之间实现双向通信的机制，是一个可以让web更加鲜活实时的关键技术。

现有的通信技术

  • XMLHttpRequest

让浏览器发往服务器的异步通信得以实现。出现早，浏览器支持情况好。HTML5新标准对其进行了二次改进使其得以跨域，跟更得心应手。

  1. AJAX(轮询) 是基于XMLHttpRequest使用setInterval,定期确认服务器的更新得以实现实时通信的方法。 缺点:即使服务器没有更新，也会进行请求，增加服务器负荷，即使更新后定时器没执行之前也无法得知，比较傻瓜式。

  2. Comet(长轮询) 只有在必要时才从服务器返回响应的方式。

  • Server-Sent Events

由服务器发往浏览器方向的推送通信，HTML5新标准。

WebSocket API使用方法

// 建立连接 
var ws = new WebSocket('ws://kunn.me:8888','可省略的子协议参数'); 
//Ps：WebSocket可以选择ws://或wss://两种协议，wss能够以TLS对通信加密，如果没有指定端口则将分别使用80与443端口,WebSocket不受同源策略现在，如有必要可以在服务器端限制连接。在函数调用后，内部将会执行握手处理，一旦建立了连接，就会触发open事件，如下

// 连接建立时的处理函数 
ws.onopen = function(event) { 

  // 向服务器发送信息
  ws.send('Hello,My name is kunn!'); 

  // 监听服务器的返回消息
  ws.onmessage = function(event) { 
    //对消息进行处理
    }; 

  // 切断ws连接
  ws.close() 

   // 监听ws关闭
  ws.onclose = function(event) { 
    //进行处理
    }; 
};

带Socket.IO的WebSocket

Socket.io是Guillermo Rauch创建的WebSocket API，Guillermo Rauch是LearnBoost公司的首席技术官以及LearnBoost实验室的首席科学家。

Socket.io使用检测功能来判断是否建立WebSocket连接，或者是AJAX long-polling连接，或Flash等。可快速创建实时的应用程序。

Socket.IO还提供了一个NodeJS API，它看起来非常像客户端API。

建立客户端Socket.io

Socket.io可以从GitHub下载，可以把socket.io.js文件包含到页面中：

<script src="http://cdn.socket.io/stable/socket.io.js"></script>


<script>
// 创建Socket.IO实例，建立连接
var socket= new io.Socket('localhost',{ 
  port: 8080 
}); 
socket.connect(); 

// 添加一个连接监听器
socket.on('connect',function() { 
  console.log('Client has connected to the server!'); 
});

// 添加一个连接监听器
socket.on('message',function(data) { 
  console.log('Received a message from the server!',data); 
});

// 添加一个关闭连接的监听器
socket.on('disconnect',function() { 
  console.log('The client has disconnected!'); 
}); 

// 通过Socket发送一条消息到服务器
function sendMessageToServer(message) { 
  socket.send(message); 
}
</script>

Socket.io简化了WebSocket API，统一了返回运输的API。传输包括：

WebSocket

Flash Socket

AJAX long-polling

AJAX multipart streaming

IFrame

JSONP polling

你还可以设置任意的Socket.IO构造器的第二个选项，选项包括：

port - 待连接的端口

transports - 一个数组，包含不同的传输类型

transportOptions - 传输的参数使用的对象，带附加属性

Socket.IO还提供了由本地WebSocket API提供的普通连接、断开连接、消息事件。Socket还提供了封装每个事件类型的方法。
