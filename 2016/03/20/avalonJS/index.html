AvalonJS

发表于 2016-03-20   |   分类于 AvalonJs   |  

一，关于AvalonJs

avalonJs是一个WEB前端的MVVM框架，由司徒正美开发和维护。它可以让你轻松实现双向绑定与数据的隔离，并且可以很好地与Jquery和平共处。

avalonJs根据兼容性和平台的不同分为三个版本：
1、兼容IE6到及其他浏览器的avalon.js。
2、只兼容到IE10和其他标准浏览器的avalon.modern.js。
3、适合运行于手机端，只是在avalon.modern.js的基础上加上了触屏事件支持的avalon.mobile.js。

二，为什么选择AvalonJs

先来看看AvalonJS的优点

1、使用简单，在HTML中添加绑定，在JS中用avalon.define定义ViewModel，再调用avalon.scan方法，它就能动了！
2、兼容到 IE6 (其他MVVM框架，KnockoutJS(IE6), AngularJS(IE9), EmberJS(IE8), WinJS(IE9) )，另有avalon.mobile，它可以更高效地运行于IE10等新版本浏览器中
3、没有任何依赖，不到5000行，压缩后不到50KiB
4、支持管道符风格的过滤函数，方便格式化输出
5、局部刷新的颗粒度已细化到一个文本节点，特性节点
6、要操作的节点，在第一次扫描就与视图刷新函数相绑定，并缓存起来，因此没有选择器出场的余地。
7、让DOM操作的代码近乎绝迹
8、使用类似CSS的重叠覆盖机制，让各个ViewModel分区交替地渲染页面
9、节点移除时，智能卸载对应的视图刷新函数，节约内存
10、操作数据即操作DOM，对ViewModel的操作都会同步到View与Model去
11、自带AMD模块加载器，省得与其他加载器进行整合。

Ps 此段出自: 知乎 作者：汤威

再来看看AvalonJs的缺点

1，对数组的过滤没有angular那么方面，avalon的过滤器只能用于ms-text, ms-html(1.5版本好了很多)
2，没有英文文档，宣传受到限制（许多人也误把我在博客园的入门教程当成官网，其实官网是在GITHUB上 http://avalonjs.github.io/）
3，OniUI的皮肤不够好看。（ps:OmiUI暂不支持1.5版本，avalonJs有AvalonUI-Bootstrap项目）

Ps 此段出自: 知乎 作者：司徒正美

三，我们来看看avalon官网上的，十秒钟看懂avalonJs，并讲解一下其中的知识要点。

<!DOCTYPE html>
<html>

<head>
    <title>first</title>
    <meta charset="UTF-8">
    <script src="//cdn.bootcss.com/avalon.js/1.5.5/avalon.js"></script>
    <script>
        var vm = avalon.define({
            $id: "test",
            name: "谦和"
        })
    </script>
</head>

<body ms-controller="test">
    <input ms-duplex="name">
    <p>Hello,{{name}}!</p>
</body>

</html>

Ps 复制到html文件在浏览器查看效果。MVVM就这么简单！

注释：
1、<body ms-controller="test">中的 ms-controller ，用来声明avalon的作用域并且需指定控制域名。类似于angularjs中的 ng-controller。
在一个HTML文件中我们可以定义多个 ms-controller。Ps： 一个控制域与一个Model相对应，当Model中的数据发生变化，与之对应的控制域的DOM会自动进行变化。
2、avalon.define()在avalon中，我们用 avalon.define({…}) 的形式来定义一个Model实例（其参数可以看做一个avalon数据对象），其中的$id是内置属性，对应所要扫描和监控的控制域名。
3、avalon使用 ms- 开头的属性。所以只要涉及到Avalon动态处理功能的，都要注意定义ms- 开头的属性。
4、ms-duplex=”name” 绑定数据到VM。 Ps：ms-duplex是avalon最重要的功能之一，是唯一一个能让试图同步到VM的指令。这里需要注意的是，它绑定的元素只是表单元素。
5、两个大括号为avalon的插值表达，与AngularJs中的使用方法一样。用于把在VM中的数据插入到HTML里。
